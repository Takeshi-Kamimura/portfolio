Case Study

株式会社サクセス（コーポレートサイト）

チーム制作でのサイト改善提案。Figma導入、デザインディレクション担当。

Role

デザイン / 情報設計 / Figma運用

Team

デザイン2・コーディング2・ディレクション1

Period

2週間

Tools

Figma / Photoshop / Illustrator / HTML / CSS

Result

クラス内 3位（6チーム中）

課題 方針 改善 成果 学び

課題

企業側からの要求点

  • 【採用エンジニア向け】明るく働きやすい印象を与えて、会社に興味を持ってもらう
  • 【企業担当向け】サービス概要や理念を一覧化し、取引につながりやすいイメージを与える
  • 社長ブログへのアクセスを増やしたいのでアピール強めに
  • 現行にない『組織』ページを作成する
  • 少しの文章とイラスト一枚で、離職率の低さや成長の度合いが伝わるようにしてほしい
  • "ゆるスタイリッシュ"なサイトを作ってほしい
  • イラストはいいが、写真はあまり使いたくない

方針（Thinking）

未経験の人に間口を広く、エンジニア志望者が「ここなら安心して応募できそう」と感じられる第一印象をつくり、採用の間口を広げる。

ターゲット

雰囲気がわからず不安な人に、安心感を届ける

  • メインターゲット：大学生〜30代前半 / 未経験〜第二新卒
  • サブターゲット：企業担当

コンセプトと表現の方向性

ゆるスタイリッシュ×手書きイラストで世界観を統一

  • 親しみやすさと信頼感を両立
  • 写真に依存しない手書きイラスト中心の表現
  • 構えずに見られ、情報が自然に入るデザイン

成果

  • クラス内3位（6チーム中）
  • Figma・Google Drive導入で制作速度・共有の質が改善
  • レスポンシブ設計まで実装
  • 興味 → 理解 → 安心へつなぐサイトを作れた

学び

  • 早期に判断基準を揃えると手戻りが減る
  • 見た目より先に情報設計が効く
  • 初めてのチーム作業で、作業分担・共有の重要さと難しさを体験できた
  • Figma・Google Driveでのチーム作業は初の試みだったが、導入してしまえば便利に進められた
  • 途中から変更ログをFigmaの別フレームに残すようにした
